The important difference

The Kaggle page labels the candidate dataset “CC0: Public Domain.” That is a license claim. But it does not say where the descriptions, notes, and image links came from, or show that the uploader could place every field under CC0. That missing chain is a provenance problem.

NosePrint’s result: inconclusive. That does not mean “definitely forbidden.” It means “we do not have enough evidence to call these rows trustworthy Real Catalog inventory.”

How the code protects the catalog

The audit command records evidence, checks the file’s schema and row count, and calculates a SHA-256 fingerprint. The import command then checks all gates again and refuses a different file.

python3 -m noseprint.catalog audit ...
# Audit verdict: INCONCLUSIVE
# exit code 2

python3 -m noseprint.catalog import ...
# Import blocked: audit verdict is not PASSED.

A passing test fixture proves the machinery works without pretending the real candidate passed. When a row is accepted, SQLite stores both its original values and cleaned values. Missing identity is rejected; missing scent notes is quarantined; repeated editions are counted as duplicates.

Quick check

Question: If someone changes only the report word from “inconclusive” to “passed,” should import start?

Reveal the answer

No. The importer independently checks every recorded gate and its evidence. One edited word cannot unlock the Real Catalog.

Why this matters for vector search

A vector database can find similar rows very quickly, but it cannot tell whether those rows were legally and honestly sourced. Trust has to be established before embeddings and nearest-neighbor search. Fast answers built from questionable facts are still questionable answers.
